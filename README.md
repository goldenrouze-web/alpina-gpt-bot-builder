# Alpina GPT Bot Builder

## Описание проекта

**Alpina GPT Bot Builder** — это Django-приложение, предоставляющее API для создания и управления чат-ботами, работающими на базе GPT.

Проект реализует конструктор ботов, где пользователь может:

* создавать ботов
* настраивать сценарии общения
* управлять шагами сценариев
* интегрировать бота с GPT API
* (опционально) подключить Telegram-бота для обработки сообщений пользователей

Основная идея проекта — предоставить гибкую систему сценариев, где каждый бот может иметь несколько сценариев взаимодействия с пользователями, состоящих из последовательных шагов.

# Основные сущности системы

## Bot

Бот представляет отдельного GPT-ассистента.

Основные поля:

* `name` — имя бота
* `description` — описание
* `owner` — владелец бота
* `telegram_token` — токен Telegram бота
* `created_at` — дата создания

---

## Scenario

Сценарий определяет логику взаимодействия пользователя с ботом.

Один бот может иметь несколько сценариев.

Поля:

* `name`
* `description`
* `bot` — бот, к которому относится сценарий

---

## Step

Шаг — это отдельный элемент сценария.

Каждый шаг содержит:

* `order` — порядок выполнения
* `prompt` — инструкция для GPT
* `next_step` — следующий шаг сценария

# API

API реализовано с использованием **Django REST Framework**.

## Bots

Работа с ботами.

| Метод  | Endpoint          | Описание       |
| ------ | ----------------- | -------------- |
| GET    | `/api/bots/`      | список ботов   |
| POST   | `/api/bots/`      | создание бота  |
| GET    | `/api/bots/{id}/` | получение бота |
| PUT    | `/api/bots/{id}/` | обновление     |
| DELETE | `/api/bots/{id}/` | удаление       |

---

## Scenarios

Работа со сценариями.

| Метод  | Endpoint               | Описание           |
| ------ | ---------------------- | ------------------ |
| GET    | `/api/scenarios/`      | список сценариев   |
| POST   | `/api/scenarios/`      | создание сценария  |
| GET    | `/api/scenarios/{id}/` | получение сценария |
| PUT    | `/api/scenarios/{id}/` | обновление         |
| DELETE | `/api/scenarios/{id}/` | удаление           |

---

## Steps

Работа с шагами сценариев.

| Метод  | Endpoint           | Описание       |
| ------ | ------------------ | -------------- |
| GET    | `/api/steps/`      | список шагов   |
| POST   | `/api/steps/`      | создание шага  |
| GET    | `/api/steps/{id}/` | получение шага |
| PUT    | `/api/steps/{id}/` | обновление     |
| DELETE | `/api/steps/{id}/` | удаление       |

# Интеграция с GPT API

Проект использует **OpenAI GPT API** для генерации ответов.

Для работы необходимо задать переменную окружения:

```
OPENAI_API_KEY
```

Пример:

```
export OPENAI_API_KEY=your_api_key
```

Взаимодействие с GPT реализовано в файле:

```
bots/gpt_service.py
```

---

# Telegram интеграция (опционально)

Проект поддерживает интеграцию с Telegram-ботом.

Используется библиотека:

```
python-telegram-bot
```

Бот принимает сообщения пользователя, отправляет их в GPT API и возвращает ответ.

Основная логика реализована в:

```
bots/telegram_bot.py
```

# Установка и запуск

## 1. Клонирование репозитория

```
git clone <repository_url>
cd alpina_gpt_builder
```

---

## 2. Создание виртуального окружения

```
python -m venv venv
```

Активировать:

Linux / Mac

```
source venv/bin/activate
```

Windows

```
venv\Scripts\activate
```

---

## 3. Установка зависимостей

```
pip install -r requirements.txt
```

---

## 4. Применение миграций

```
python manage.py migrate
```

---

## 5. Создание администратора

```
python manage.py createsuperuser
```

---

## 6. Запуск сервера

```
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```
http://127.0.0.1:8000
```

API:

```
http://127.0.0.1:8000/api/
```

Админка:

```
http://127.0.0.1:8000/admin
```
